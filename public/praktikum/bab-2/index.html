📑 Sub Bab

Setelah berhasil mengontrol keluaran (output) dasar pada Bab 1, pada Bab 2 ini kita akan membuat sistem menjadi lebih interaktif. Fokus utama praktikum kali ini adalah mempelajari bagaimana mikrokontroler “merasakan” dan merespons kondisi dari luar.

Berikut adalah gambaran utama dari kegiatan praktikum yang akan Anda pelajari dan praktikkan:

  • 🔘 Pembacaan Input Digital (Push Button) Anda akan merangkai sakelar tekan (push button) dan memprogram NodeMCU untuk membaca status logika digital (HIGH/LOW atau 1/0). Tahap ini penting untuk memahami bagaimana mikrokontroler menerima instruksi langsung dari intervensi manusia.

  • 🎛️ Implementasi Input Analog Berbeda dengan sinyal digital yang hanya memiliki dua kondisi (nyala/mati), di sini Anda akan bereksperimen dengan pin analog (A0) untuk membaca nilai rentang yang bervariasi (misalnya menggunakan potensiometer atau sensor cahaya/LDR). Anda akan memantau perubahan nilai ini secara real-time melalui Serial Monitor.

  • ⚙️ Pengenalan Sinyal PWM (Pulse Width Modulation) Anda akan mengaplikasikan konsep PWM untuk mengontrol perangkat keluaran secara lebih presisi. Alih-alih hanya menyalakan atau mematikan LED, Anda akan memprogram mikrokontroler untuk mengatur tingkat kecerahan LED secara bertahap (fading).

  • 🧠 Logika Percabangan (If-Else) Tahap integrasi logika. Anda akan menulis kode pemrograman bersyarat (conditional statement) agar mikrokontroler dapat mengambil keputusan secara mandiri. Contohnya: “Jika tombol ditekan, maka LED menyala; jika tidak, LED mati.”

Tujuan Praktikum: Di akhir Bab 2, Anda diharapkan mampu merangkai dan mengintegrasikan komponen input digital maupun analog, serta menguasai penulisan logika percabangan agar perangkat keras mampu berinteraksi dan merespons kondisi sekitarnya dengan tepat.
